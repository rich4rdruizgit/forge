## Purpose

Closes a completed feature cycle: validates all required phases are `✅ Aprobado` (SPIKE is optional — if skipped, it does not need approval), generates INDEX.md from all approved artifacts, moves all artifacts (including INDEX.md) from `.forge/features/activo/{slug}/` to `.forge/features/closed/{slug}/`, adds a history row to FORGE.md, and resets the active cycle to null.

---

## Preconditions

- Active feature exists (`feature` ≠ null in FORGE.md)
- ALL required phases = `✅ Aprobado`: SPEC, BUILD, VERIFY must be approved. SPIKE is optional — if it was used (status is not `⏳ Sin iniciar`), it must also be `✅ Aprobado`. If SPIKE was skipped (`⏳ Sin iniciar`), it does not block closing.
- If ANY required phase is not approved: STOP, list unapproved phases

---

## Forge Runtime

→ Execute `_shared/forge-runtime.md` steps R1–R4 before any skill-specific logic.

---

## Execution Steps

ORDER IS CRITICAL — do not reorder these steps.

### Step C1 — Verify activo folder exists
Confirm `.forge/features/activo/{slug}/` exists and contains files.
If not found: STOP. Error E601: "No encontré `.forge/features/activo/{slug}/`. ¿Fue movida manualmente?"

### Step C2 — Generate INDEX.md

Read ALL approved artifacts from `.forge/features/activo/{slug}/`:
- SPIKE.md (if exists — optional)
- SPEC.md
- VERIFY.md
- TRACEABILITY.md

Also read if they exist (generated by forge approve):
- VALIDATION-SPIKE.md
- VALIDATION-SPEC.md
- VALIDATION-BUILD.md
- VALIDATION-VERIFY.md

Generate INDEX.md in `.forge/features/activo/{slug}/INDEX.md` following the INDEX.md template.

Populate these sections:
- **Acceptance Criteria**: Extract all ACs from SPEC.md §1 Requirements
- **Domain Events**: Extract all events from SPEC.md §2 Domain Model
- **UI Contract Summary**: Extract UI states/screens from SPEC.md §5 UI Contract
- **Architecture Decisions**: Extract all decisions from SPEC.md §4 Decisions
- **Key Files**: Extract file list from SPEC.md §3 Architecture
- **Test Coverage Metrics**: Extract from VERIFY.md (total tests, coverage %)
- **Validation Scores**: Extract final scores from each VALIDATION-*.md file (if present)
- **Traceability Summary**: Extract AC→Event→UI State→Test→Implementation counts from TRACEABILITY.md (if present)
- **Patterns & Conventions**: Notable patterns used (from SPEC.md §3 Architecture)
- **Addenda**: Extract any addenda from SPEC.md Addenda section (if present)
- **Tags**: Auto-generate searchable tags from stack, feature name, patterns used

### Step C2.5 — Knowledge Extraction

After generating INDEX.md and BEFORE archiving, extract knowledge for KNOWLEDGE.md.

#### Step K1 — Read artifacts
Read: approved SPEC, VERIFY report, TRACEABILITY.md, INDEX.md

#### Step K2 — Extract candidates
For each KNOWLEDGE.md section, identify candidates:
- **Patrones Establecidos**: ¿Se usó un patrón no registrado?
- **Decisiones Técnicas Globales**: ¿Alguna decisión afecta más que esta feature?
- **Contratos Conocidos**: ¿Nuevo endpoint, componente, o interfaz?
- **Componentes Reutilizables**: ¿Se creó algo diseñado para reuso? (check SPEC section 5b)
- **Errores y Lecciones**: ¿Algo inesperado que un dev futuro debería saber?
- **Módulos Tocados**: SIEMPRE actualizar

#### Step K3 — Present to dev
Present each candidate with concrete format. The dev approves, edits, or rejects EACH ONE individually.

#### Step K4 — Write approved entries
ONLY write entries approved by the dev. Add to the corresponding section of KNOWLEDGE.md.
Update "Última actualización" header.

**Rule**: NEVER write to KNOWLEDGE.md without explicit dev approval per entry.

### Step C3 — Create closed folder if needed
Check if `.forge/features/closed/` exists. If not: create it. Not an error.

### Step C4 — Handle slug collision
Check if `.forge/features/closed/{slug}/` already exists.
If exists: use `{slug}-{YYYY-MM-DD}` as target folder name (append today's date).

### Step C5 — Move all artifacts
Move all files from `.forge/features/activo/{slug}/` to `.forge/features/closed/{slug}/` (or slug with date suffix).
This includes SPIKE.md (if exists), SPEC.md, VERIFY.md, TRACEABILITY.md, INDEX.md, and any VALIDATION-*.md files.
This is a MOVE, not a copy. The activo/{slug}/ folder must be empty afterward.

### Step C6 — Remove activo slug folder
Remove the now-empty `.forge/features/activo/{slug}/` directory.

### Step C7 — Update FORGE.md history table (DO THIS BEFORE RESETTING ACTIVE STATE)
Add a row to the history table:
| {feature} | {slug} | {azure_story or —} | {YYYY-MM-DD} | ✅ Cerrada | {overall validation score or —} |

The overall validation score is the average of all phase validation scores, or `—` if no validation was run.

### Step C8 — Reset FORGE.md active cycle
Reset the YAML active cycle block:
  feature:       null
  slug:          null
  azure_story:   null
  fase_actual:   null

Reset the phase table — all 4 rows:
  | Fase | Archivo | Estado | Validation Score |
  | SPIKE  | —       | ⏳ Sin iniciar | — |
  | SPEC   | —       | ⏳ Sin iniciar | — |
  | BUILD  | —       | ⏳ Sin iniciar | — |
  | VERIFY | —       | ⏳ Sin iniciar | — |

## Rules

- **NUNCA cerrar si cualquier fase no está ✅ Aprobado — sin excepciones**
- SIEMPRE generar INDEX.md (Step C2) ANTES de mover artefactos (Step C5)
- SIEMPRE actualizar el historial (Step C7) ANTES de resetear el estado activo (Step C8)
- SIEMPRE mover los artefactos (no copiar) — la carpeta activo/{slug}/ debe quedar vacía
- NUNCA borrar artefactos — van a closed/, no a la papelera
- NUNCA resetear FORGE.md si la operación de mover falló

## Engram Memory

After archiving, save the completed feature to Engram so future features can learn from it.

**If Engram tools are available** (`mem_save`), execute:

Call `mem_save` with:
- **title**: `Feature cerrada: {feature-name}`
- **type**: `project`
- **topic_key**: `.forge/features/{slug}`
- **content**:
  ```
  **Feature**: {feature-name}
  **Slug**: {slug}
  **Stack**: {stack value from .forge/config.yaml}
  **Fecha de cierre**: {today's date}
  **Validation Score**: {overall validation score or N/A}

  **Acceptance Criteria (de SPEC.md §1)**:
  {list all ACs from the closed feature's SPEC.md §1 Requirements}

  **Domain Events (de SPEC.md §2)**:
  {list all domain events from the closed feature's SPEC.md §2 Domain Model}

  **Architecture Decisions (de SPEC.md §4)**:
  {list all decisions from the closed feature's SPEC.md §4 Decisions}

  **Key Files (de SPEC.md §3)**:
  {list of files/modules affected from SPEC.md §3 Architecture}

  **Decisiones clave**:
  {any notable architectural or design decisions found in SPEC.md §4}

  **Patterns**:
  {patterns and conventions used}

  **Tags**: {comma-separated tags from INDEX.md}
  ```

If Engram is not available, skip silently — do NOT block or warn. Forge works without Engram.
